# Анализ влияния нормализации временных рядов на модели разных классов
Работу выполнил студент 1-го курса направления "Искусcтвенный Интеллект" Железнов Кирилл Игоревич
## Описание
В проекте исследуется влияние нормализации для разных классов моделей:
- Naive
- SeasonalNaive
- AutoETS
- AutoTheta
- CatBoostRegressor
- PatchTST
Используется датасет M4 Monthly (https://forecastingdata.org/ https://zenodo.org/records/4656480)

Основная цель - проверить как нормализация влияет на метрики прогнозов.

## Проверяемая гипотеза (условие)
Нормализация по рядам особенно важна для глобальных нейронных моделей, но влияет неоднозначно для моделей из разных классов.
- Зафиксировать бейзлайны и две глобальные модели для основных экспериментов (например, CatBoost и PatchTST).
- Сравнить качество моделей (бейзлайнов тоже) с применением StandardScaling и без его применения.
- Сравнить качество моделей с применением не менее двух других вариантов скейлинга (QuantileScaling, RobustScaling).
- Отдельно проанализировать, одинаково ли влияет нормализация на модели разных классов.

## Структура проекта
```
/data - директория с исходными данными (/data/m4_monthly_dataset.tsf)  
/results - директория с результатами экспекриментов и анализа  
/results/dataset.csv - выбранные в ходе экспериментов ряды
/results/forecasts.csv - прогнозы всех моделей
/results/metrics.csv - значения метрик
/results/analysis_results.ipynb - анализ результатов
/src - директория со своими функциями для экспериментов
/src/__init__.py
/src/dataloader.py - функции загрузки и преобразования данных
/src/models.py - обучение моделей, прогноз, метрики
config.py - параметры проекта
run_experiments.py - запуск экспериментов
requirements.txt - зависимости
README.md
```

## Использование
1. Установите зависимости `pip install -r requirements.txt`
2. Положите датасет m4_monthly_dataset.tsf в папку data/ (если другой путь - укажите полный путь в конфиге)
3. Запустите эксперименты `python run_experiments.py`
4. После появятся файлы `results/dataset.csv`, `results/forecasts.csv`, `results/metrics.csv`
5. Откройте `results/analysis_results.ipynb` для визуализации результатов
